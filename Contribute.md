## 🚀 Contribution Workflow

### 1️. Fork the Repository

Click **Fork** on top right

### 2️. Clone Your Fork

```bash
git clone https://github.com/<your-username>/teach-back-session.git
cd teach-back-session
```

### 3️. Create a Branch

```bash
git checkout -b your-name-feature
```


### 4. Commit Properly

```bash
git add .
git commit -m "Added: short meaningful message"
```
 Good examples:

* `Fixed: typo in branching tutorial`
* `Improved: merge conflict explanation`

---

### 5. Push Changes

```bash
git push origin your-name-feature
```

---

### 6. Create a Pull Request (PR)

* Go to your fork on GitHub
* Click **Compare & Pull Request**
* Add:

  * Title
  * Description (what + why)

## ⚠️ Rules & Best Practices

* ✔️ Keep commits small and meaningful
* ✔️ Don’t push directly to `main`
* ✔️ Always create a branch
* ✔️ Write clear PR descriptions
* ✔️ Be respectful in reviews

---

## Special Practice Tasks

To simulate real-world development:

* Break things and fix them
* Collaborate with others
